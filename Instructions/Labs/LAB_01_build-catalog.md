# 实验室 1：在 Microsoft Entra 权利管理中构建目录

## 实验室方案

在一家中型软件开发公司，IT 部门决定实施 Microsoft Entra 进行权利管理。 其主要目标是简化整个组织对资源和应用程序的访问。 使用 Microsoft Entra，他们可以根据角色或项目定义访问包，从而简化授予或撤消访问权限的流程。 例如，当新的开发人员加入项目时，IT 部门可以通过将其分配到相应的访问包来轻松为其提供必要的访问权限。 这不仅可以节省时间，还能降低未经授权访问的风险。 此外，Microsoft Entra 的定期访问评审可确保只有正确的人员有权访问敏感资源。 在实施实验室中，IT 团队将设置各种访问包，定义用于自动访问分配和撤销的策略，并进行模拟访问评审。

## 目标

完成本实验室后，你将能够：

- 构建目录。
- 配置访问包。
- 将访问包部署到用户。
- 接受作为用户的权利并确认资源访问权限。
- 撤销对包的访问权限。

## 实验室教学设置
  - **预计用时**：30 分钟

### 练习 1：为销售团队创建目录

#### 任务 1 - 创建目录

1. 在 `https://Entra.Microsoft.com` 启动 Microsoft Entra 管理中心。

1. 在左侧菜单中，导航到“**标识治理**”，然后导航到“**权利管理**”。

1. 从菜单中选择“**目录**”。

 ![“创建新目录屏幕“的屏幕截图，其中突出显示“新建目录”按钮。  包含“名称”、“说明”和“启用”的字段。](./Media/create-new-catalog.png)

1. 选择屏幕顶部的“**+ 新建目录**”。

1. 使用以下值命名和说明“**新建目录**”：

  | 字段 | 值 |
  | :---  | :---  |
  | 名称  | `catSales` |
  | 说明 | `Use this catalog to assign resources for memebers of the Sales team.` |
  | 已启用 | 是 |
  | 已为外部用户启用 | 否 |
  | | |

1. 选择**创建**。

#### 任务 2：将资源添加到目录

1. 如果还没有访问那里，请导航到“**Microsoft Entra 管理中心**”、“**标识治理**”、“**权利管理**”，然后导航到“**目录**”页。

1. 选择我们在上一个任务中创建的 **catSales**。

1. 从菜单中选择“**资源**”。

1. 接下来，从页面顶部选择“**+ 添加资源**”。

 ![将资源添加到目录用户界面的屏幕截图。 可以添加 Teams、应用程序、SharePoints 和其他项。](./Media/add-resources-to-catalog.png)

1. 使用屏幕顶部的选择器，添加以下资源：

  | 资源类型 | 值 |
  | :---  | :---  |
  | + 组和团队  | 销售和市场营销、美国销售 |
  | + 应用程序 | LinkedIn |
  | + SharePoint 站点 | 销售和市场营销、美国销售 |
  | | |

1. 选择“添加”按钮。

#### 任务 3：创建新用户以接收权利

1. 如果还没有访问那里，请导航到“**Microsoft Entra 管理中心**”。

1. 从左侧菜单中选择“**标识**”、“**用户**”，然后选择“**所有用户**”。

1. 选择页面顶部的“**+ 新建用户**”。

1. 在“**基本信息**”页上填写值：

  | 字段 | 值 |
  | :---  | :---  |
  | 用户主体名称  | `ChrisGr` |
  | 显示名称 | `Christopher Green` |
  | 自动生成密码 | 选中 |
  | 已启用帐户 | 选中 |
  | | |

1. 将“密码”复制并粘贴到记事本等安全位置（在本实验室稍后需要密码）。

1. 选择“属性”选项卡。

1. 在“属性”屏幕底部，设置“**使用情况位置 = 美国**”。

1. 选择“**查看 + 创建**”、然后选择“**创建**”。

#### 任务 4：生成访问包

1. 在 Microsoft Entra 管理中心，依次选择“**标识治理**”和“**权利管理**”。

1. 从“权利管理”菜单中选择“**访问包**”。

1. 选择屏幕顶部的“**+ 新建访问包**”。

1. 输入请求的值：

  | 字段 | 值 |
  | :---  | :---  |
  | 名称  | `pckSales` |
  | 显示名称 | `Use this access package to assign resources to members of the Sales team.` |
  | 目录 | catSales |
  | | |

  **备注** - 必须选取我们在上一个任务中创建的 catSales 目录。 这将提供可用于在此包中分配的资源列表。  有一个常规包被列为默认包。  如果意外选择了此项，则不会看到任何可用资源。

1. 选择“资源角色”选项卡。

1. 从 catSales 目录的项中，选择想要在其访问包中提供的资源。 然后使用“**选择角色**”下拉列表设置下表中列出的角色。

  | 资源类型 | 值 | 角色 |
  | :---  | :---  | :--- |
  | + 组和团队  | 销售和营销 | 成员 |
  | + 应用程序 | LinkedIn | msiam_access |
  | + SharePoint 站点 | 美国销售 | 美国销售成员 |
  | | |

1. 设置“**选择角色**”下拉列表，将每个项的角色设置为“**成员**”。

1. 使用“**下一步: 请求 >**”转到“请求”选项卡。

1. 对于“**可请求访问的用户**”，选择选项“**无 (仅限管理员直接分配)”。

1. 将“**启用**”设置为“**是**”。

1. 使用屏幕顶部的标签移动到“**生命周期**”选项卡。

1. 选择要设置包的生命周期的值：

  | 字段 | 值 |
  | :---  | :---  |
  | 访问包分配过期  | 天数 |
  | 分配在以下时间后过期 | 30 |
  | 用户可以请求特定时间线 | 否 |
  | 需要访问评审 | 否 |
  | | |

1. 在屏幕底部选择“**查看 + 创建**”。

1. 评审在“审阅 + 创建”屏幕上选择的值。

1. 选择“**创建**”以生成访问包。

#### 任务 5：将包分配给 Christopher

1. 请确保处于 **Microsoft Entra 管理中心**、**标识治理**、**权利管理**，并打开“**访问包**”菜单。

1. 选择我们在上一个任务中创建的 **pckSales**。

1. 在菜单中选择“**分配**”。

1. 从屏幕顶部，选择“**+ 新建分配**”。

1. 对于“**选择策略**”，请使用下拉列表中提供的“**初始策略**”。

1. 确保“**已在我的目录中的用户**”已标记。

1. 从对话中选择“**添加用户**”项。

1. 在用户列表中查找 **Christopher Green**。  勾选其名称旁边的框。  然后选择屏幕底部的“**选择**”按钮。

1. 将其余值保留为默认值。

1. 选择页面底部的“添加”按钮****。

#### 任务 6：检查是否添加了 Christopher Green

1. 在浏览器上打开“**新 InPrivate 窗口**”。

1. 连接到 **Microsoft Entra管理中心**，地址为 `https://entra.microsoft.com`。

1. 使用之前创建的 Chistopher Green 帐户和密码登录到站点。

1. 系统会提示你更改密码。  设置新密码并将其记录在记事本等工具中以供将来使用。

1. 依次选择“**标识**”、“**用户**”、“**所有用户**”和“**Christopher Green**”。

1. 从左侧菜单中选择“**组**”。

1. 根据访问包，确认你已获得对“**销售和市场营销**”组的访问权限。

1. 在左侧菜单中选择“**应用程序**”。

1. 确认 **LinkedIn**是你的已分配应用程序。

#### 任务 7：挑战 - 对访问包的动态更改

  **备注** - 此任务没有分步说明。 为你提供一组任务，可以参考上述先前步骤，以提醒你在哪里进行特定更改。

- 请确保以管理员帐户身份登录到 **Microsoft Entra 管理中心**。
- 打开 **pckSales** 访问包。
- 转到”**资源角色**“，选择移除“**销售和市场营销**”组，改为添加“**美国销售**”组。
- 使用“**分配**”选项卡“**重新处理**”分配。
- 以 Christopher Green 的身份注销并重新登录。  请注意，组分配已更改。  快速简单。
- 通过删除 Christopher Green 的分配来撤销访问权限。
